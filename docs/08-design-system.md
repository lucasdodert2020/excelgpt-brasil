# Design System

## Objetivo

Definir uma direção visual inicial para um SaaS profissional, confiável e focado em produtividade.

## Personalidade Visual

- Profissional.
- Limpa.
- Moderna.
- Técnica sem parecer complexa.
- Associada a produtividade, planilhas e inteligência.

## Direção de Interface

- Layouts claros e funcionais.
- Componentes densos o suficiente para uso diário.
- Boa leitura de fórmulas e blocos de código.
- Respostas com hierarquia visual entre fórmula, explicação e observações.
- Evitar aparência de ferramenta genérica de chat.

## Cores Iniciais

Paleta a ser refinada futuramente:

- Verde principal associado a Excel e produtividade.
- Cinza neutro para estrutura.
- Azul discreto para ações e links.
- Amarelo ou laranja suave para alertas de compatibilidade.
- Vermelho controlado para erros.

## Tipografia

- Fonte sans-serif moderna para interface.
- Fonte monoespaçada para fórmulas.
- Hierarquia clara entre títulos, texto, labels e respostas.

## Componentes Previstos

- Botão primário.
- Botão secundário.
- Campo de chat.
- Seletor de versão.
- Card de resposta.
- Bloco de fórmula.
- Badge de compatibilidade.
- Indicador de plano e limite.
- Modal de upgrade.
- Lista de histórico.

## Acessibilidade

- Contraste adequado.
- Navegação por teclado.
- Labels claros.
- Estados de foco visíveis.
- Fórmulas copiáveis sem depender apenas de seleção manual.
